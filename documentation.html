Overview

SmartHarvest AI is an end-to-end machine learning system that predicts district-level crop yield and a composite Climate Risk Index for 50 Indian districts and 10 major crops, using 8,000+ records spanning 2005–2023.

The system has three layers: an offline training pipeline (Python), a REST API that serves the trained model, and this static frontend, which works standalone in a demo mode and upgrades automatically once a real backend is connected.

Problem statement

Indian agriculture is highly sensitive to rainfall variability and temperature extremes, yet accessible, district-level yield forecasting and risk-scoring tools remain limited for planning and advisory use.

Objectives

  • Predict crop yield (t/ha) and total production from routinely available agro-climatic inputs.
  • Quantify climate risk as a single, interpretable composite index.
  • Compare multiple ML approaches and select the best-performing model objectively.
  • Package the system as a deployable API and a usable web interface.

System architecture

Static frontend → API Gateway → Flask app (EC2/Lambda) → pickled XGBoost model + scaler → response. Training happens offline and produces the model artifacts consumed at inference time. See the architecture diagram on the home page.

Methodology

Machine learning pipeline

  1. Data generation / ingestion (data/generate_dataset.py)
  2. Preprocessing — imputation, encoding, scaling (src/preprocessing.py)
  3. Exploratory data analysis (src/eda.py)
  4. Model training — 5 algorithms (src/models.py, src/lstm_model.py)
  5. Climate Risk Index computation (src/risk_index.py)
  6. Evaluation and best-model selection
  7. API serving (backend/app.py)

Dataset

8,000 rows across 50 districts, 10 crops, and 19 years (2005–2023). Seven input features: Rainfall, Temperature, Humidity, NDVI, Soil Moisture, Area, Crop, Year. Two targets: Production and Yield.

District,Year,Crop,Rainfall,Temperature,NDVI,Soil_Moisture,Humidity,Area,Production,Yield

Algorithms & model comparison

Linear Regression, Gradient Boosting, Random Forest, XGBoost, and a Bidirectional LSTM (rice time series) were trained on an identical 80/20 split. Full metrics are on the Model Comparison page — XGBoost was selected for production with R² = 0.931.

API reference

POST /api/predict

{ "crop": 0, "district": 12, "rainfall": 850, "temperature": 27, "humidity": 65, "ndvi": 0.62, "soil_moisture": 45, "area": 8000, "year": 2024 }

Response:

{ "predicted_yield": 2.84, "estimated_production": 22720, "risk_index": 0.32, "risk_label": "Low Risk", "factors": { ... } }

GET /api/health

Returns { "status": "ok", "model": "loaded", "version": "1.0.0" } — used by the frontend to decide live vs. demo mode.

GET /api/districts, GET /api/model-stats, GET /api/crops

Return pre-computed district risk scores, model comparison metrics, and the crop lookup table respectively.

Deployment guide

Frontend — GitHub Pages

git push origin main # Settings → Pages → Deploy from branch → main → /root

Frontend — Netlify / Vercel

Drag-and-drop the project folder, or connect the GitHub repo. No build step required — it's static HTML/CSS/JS.

Backend

Deploy backend/app.py (Flask) to Render, Railway, or AWS EC2/Lambda + API Gateway. Once live, set API_URL in js/config.js — no other frontend changes needed.
