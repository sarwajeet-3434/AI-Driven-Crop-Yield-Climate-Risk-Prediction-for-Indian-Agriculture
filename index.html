About the project

Yield forecasting built on the same signal a farmer already reads — vegetation, rain, and heat

SmartHarvest AI was built as a final-year engineering project to demonstrate a full applied ML lifecycle: synthetic-but-realistic agro-climatic data generation, exploratory analysis, model comparison, a composite risk index, and a production-shaped API — not just a notebook.

The system ingests seven agro-climatic inputs (rainfall, temperature, humidity, NDVI, soil moisture, area, year) per district-crop-year and outputs predicted yield, estimated production, and a Climate Risk Index in one call.

Problem statement

Indian agriculture is highly climate-sensitive, yet district-level yield forecasting tools remain scarce for smallholder planning. This project builds a reproducible pipeline that predicts yield and quantifies climate risk from routinely available agro-climatic variables.

Capabilities

What the system does

Seven inputs in, four decisions out — yield, production, risk, and a plain-language recommendation.

Yield & production forecasting

Predicts tonnes-per-hectare yield and estimated total production for any district-crop-year combination in the trained range.

Composite Climate Risk Index

Combines rainfall variability, NDVI stress, and temperature anomaly into a single 0–1 risk score with Low / Medium / High labels.

Five-model comparison

Linear Regression, Random Forest, Gradient Boosting, XGBoost, and a Bidirectional LSTM benchmarked on identical splits.

District risk heatmap

All 50 districts scored and color-coded so climate-vulnerable regions are visible at a glance.

Backend-agnostic frontend

Ships with a self-contained demo prediction engine and calls a real Flask/FastAPI backend automatically once one is configured — the UI never breaks.

Cloud-ready deployment

Designed for AWS EC2 / Lambda + API Gateway + S3, or any container host — documented step by step.

Future scope

Where this goes next

Satellite NDVI ingestion

Replace synthetic NDVI with live Sentinel-2 / MODIS feeds via the Copernicus or Google Earth Engine APIs.

Sub-district granularity

Extend from district-level to block/tehsil-level predictions using higher-resolution weather grids.

Farmer-facing mobile app

Progressive web app with offline caching for low-connectivity rural use, building on this PWA-ready frontend.

Ensemble + uncertainty bounds

Stack XGBoost with the Bi-LSTM and report prediction intervals, not just point estimates.

Government advisory integration

Feed high-risk district alerts into existing state agriculture department dashboards.
